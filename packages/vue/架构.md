# 导出组件设计
使用上面设计的hooks

支持WorkerPoll组件
需要使用外层组件来管理(类似于react的Privider，填写线程数和配置)
然后再使用子组件渲染(如果不使用Privider的话, 就是单线程Worker)

子组件和WorkerPoll管理Privider都能填写渲染相关配置。但是, 使用了WorkerPoll管理Privider，
子组件的MarkdownRenderer相关属性就会失效(所以子组件那些配置都是存在默认配置-方便不写)
Privider是一定要写配置的。

组件被销毁需要释放线程

渲染中需要有loading(自行使用svg动态)
然后, 渲染如果报错，组件需要做错误边界。给提醒.

渲染组件 & WorkerPollPrivider需要支持这样的api能够用户自行替换渲染节点(同理hooks也需要支持)
```
 <MarkdownRenderer
  components={{
    // Map `h1` (`# heading`) to use `h2`s.
    h1: 'h2',
    // Rewrite `em`s (`*like so*`) to `i` with a red foreground color.
    em(props) {
      const {node, ...rest} = props
      return <i style={{color: 'red'}} {...rest} />
    }
  }}
/>
```
